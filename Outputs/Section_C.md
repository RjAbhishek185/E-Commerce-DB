## Q13. Count the total number of orders in the `orders` table.

### Objective

Determine how many orders are currently stored in the `orders` table.

### SQL Query

```sql
SELECT COUNT(*) AS total_orders
FROM orders;
```

### Explanation

* `COUNT(*)` counts every record present in the table.
* Each row represents a single customer order.
* `AS total_orders` gives a descriptive name to the output column.
* `FROM orders` specifies the table being queried.

### Business Insight

The total order count gives a quick overview of customer activity. It is one of the basic metrics used to evaluate sales performance and monitor business growth.

## Q14. Find the total revenue (SUM of `total_amount`) from all Delivered orders.

### Objective

Calculate the revenue earned only from orders that have been successfully delivered.

### SQL Query

```sql
SELECT SUM(total_amount) AS total_revenue
FROM orders
WHERE status = 'Delivered';
```

### Explanation

* `SUM(total_amount)` calculates the combined value of all order amounts.
* `WHERE status = 'Delivered'` filters the data to include only completed deliveries.
* `AS total_revenue` labels the resulting value for better readability.

### Business Insight

Revenue from delivered orders reflects the actual income generated by the business since cancelled or pending orders are excluded.

## Q15. Calculate the average `unit_price` of products in each category.

### Objective

Find the average selling price for products belonging to each category.

### SQL Query

```sql
SELECT category,
       AVG(unit_price) AS average_price
FROM products
GROUP BY category;
```

### Explanation

* `AVG(unit_price)` computes the average product price.
* `GROUP BY category` groups products based on their category before calculating the average.
* `AS average_price` provides a meaningful column name.

### Business Insight

Comparing average prices across categories helps identify which categories contain higher-value or lower-value products, supporting pricing and marketing decisions.

## Q16. Display the number of orders and total revenue for each order status. Sort the results by total revenue in descending order.

### Objective

Generate a summary showing order count and revenue for every order status, ordered from highest to lowest revenue.

### SQL Query

```sql
SELECT status,
       COUNT(*) AS total_orders,
       SUM(total_amount) AS total_revenue
FROM orders
GROUP BY status
ORDER BY total_revenue DESC;
```

### Explanation

* `COUNT(*)` returns the number of orders for each status.
* `SUM(total_amount)` calculates total revenue within each group.
* `GROUP BY status` creates separate groups for each order status.
* `ORDER BY total_revenue DESC` arranges the output from the highest revenue to the lowest.

### Business Insight

This report helps compare the financial contribution of different order statuses and highlights whether pending or cancelled orders are affecting business performance.

## Q17. Find the most expensive (MAX) and cheapest (MIN) product in each category.

### Objective

Identify the highest-priced and lowest-priced product available in every category.

### SQL Query

```sql
SELECT category,
       MAX(unit_price) AS most_expensive,
       MIN(unit_price) AS cheapest
FROM products
GROUP BY category;
```

### Explanation

* `MAX(unit_price)` returns the highest product price within each category.
* `MIN(unit_price)` returns the lowest product price within each category.
* `GROUP BY category` performs these calculations separately for every category.

### Business Insight

Knowing the price range of each category helps businesses understand their product positioning and supports pricing, inventory, and promotional strategies.

## Q18. Display categories where the average product price is greater than ₹2000 using the `HAVING` clause.

### Objective

Retrieve only those product categories whose average price is above ₹2000.

### SQL Query

```sql
SELECT category,
       AVG(unit_price) AS average_price
FROM products
GROUP BY category
HAVING AVG(unit_price) > 2000;
```

### Explanation

* `AVG(unit_price)` calculates the average price for each category.
* `GROUP BY category` groups similar products together.
* `HAVING AVG(unit_price) > 2000` filters the grouped results based on the calculated average value.

### Business Insight

This query identifies premium-priced categories, helping management focus on high-value products and design suitable pricing or marketing strategies.
